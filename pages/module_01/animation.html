keyframes

The @keyframes tule specifies the animation code.

You must specify a name and then declare attributes to create a animation.

One of many methods to create a animation with CSS, using @keygrame, using attributes from and to.

  • from: where it start's
  • to: where it end's

CSS Code

@keyframes example {
  from {
    width: 5%;
  }
  to {
    width: 100%;
  }
}
div#example-1 {
  background-color: red;
  height: 100px;
  animation-name: example-1;
  animation-duration: 3s;
  animation-delay: 2s;
}

HTML Code

<div id='example-1'>
</div>

Result

animation-name

This attribute link's to @keyframe name defined before.

animation-duration

This attribute defines the duration's animation.

animation-delay

This attribute defines animation's initiation. If set to 2 seconds, then animation will start after two seconds.

animation-iteration-count

This attribute define's how many times the animation will repeat.

animation-fill-mode

The animation-fill-mode property specifies a style for the element when the animation is not playing (before it starts, after it ends, or both).

CSS animations do not affect the element before the first keyframe is played or after the last keyframe is played. The animation-fill-mode property can override this behavior.

webkits

Some browsers doesn't support keyframes, because it's an actual attribute, so, maybe some browsers don't support it.

To overcome this, you must use webkits, that was the animation manner.

CSS Code

@-webkit-keyframes example-1 {
  from {
    width: 5%;
  }
  to {
    width: 100%;
  }
}
div#example-8 {
  background-color: red;
  height: 100px;
  -webkit-animation-name: example-1;
  -webkit-animation-duration: 3s;
  -webkit-animation-delay: 1s;
  -webkit-animation-iteration-count: infinite;
}

HTML Code

<div id='example-8'>
</div>

Result
